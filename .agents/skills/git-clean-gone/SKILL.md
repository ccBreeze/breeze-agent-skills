# Git Clean Gone

## 目标

安全清理 upstream 已移除的本地陈旧分支，同时避免误删当前分支、默认分支和未合并工作。

## 工作流

1. 识别 gone 分支

- 先执行同步信息所需的检查，如 `git fetch --prune`。
- 通过 `git branch -vv` 或等价方式识别 upstream 标记为 `gone` 的本地分支。
- 输出候选列表前，先确认当前分支与默认分支名称。

2. 做删除前安全检查

- 永远不要删除当前分支。
- 永远不要删除 `main`、`master` 或仓库默认分支。
- 仅自动删除已经合并进默认分支的 gone 分支。
- 如果分支未合并、状态不明或仍可能有价值，保留并在结果中说明原因。

3. 清理分支

- 默认使用 `git branch -d` 做安全删除。
- 不要使用 `git branch -D`，除非用户明确要求强制清理。
- 删除后再次检查，确认 gone 分支列表已经更新。

4. 返回结果

- 列出已删除分支与跳过分支。
- 对跳过项说明具体原因，例如“未合并”、“当前分支”或“默认分支保护”。
