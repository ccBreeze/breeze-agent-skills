# 本仓库补充约定（非官方）

本文档仅适用于 `breeze-agent-skills` 仓库，用来补充该仓库对 commit message 的本地格式要求。
这些规则不是 `git-commit` 技能的通用/官方规范，只有在处理本仓库提交时才应采用。

## Commit Message 格式

本仓库的 commit title 与 body 默认使用简体中文表达。
若用户未另行指定，避免使用英文标题或英文正文描述改动。

## Commit Body 格式

当提交信息需要正文时，正文使用 `- ` 开头的列表项表达，每行一条完整信息。
不要写成普通段落，也不要混用段落和列表。
不同列表项之间不要额外插入空行，保持紧凑的连续列表。

推荐形式：

```text
<type>(<scope>): 中文标题

- 第一条结果或改动
- 第二条结果或改动
- 第三条结果或改动
```

示例：

```text
fix(dev): 端口占用时快速失败并输出清理命令

- 为共享 Vite 开发服务启用 strictPort，避免端口被占时静默切换
- 在 dev 启动失败后打印端口清理命令，并从各应用配置动态解析端口
```

## 使用时机

- body 中的每一条说明都使用 `- ` 开头的列表项单独表达
- 如果本次提交不需要 body，可以只写标题行，不必补空正文
